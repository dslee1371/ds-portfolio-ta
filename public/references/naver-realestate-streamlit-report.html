네이버 부동산 스크래퍼 & Streamlit 대시보드

데이터 파이프라인 구조, 자동화 워크플로우, 그리고 시각화 대시보드 화면을 정리했습니다.

GitHub 레포지토리

전체 파이프라인은 dslee1371/naver-realestate-streamlit 에 공개되어 있습니다. infra/ 디렉터리에는 GitOps 매니페스트와 Helm 차트가 정리되어 있습니다.

ETL & 모니터링 구성

  • Requests + Playwright 조합으로 30분 간격 데이터 적재, 실패 시 재시도 파이프라인 구성.
  • Airflow DAG에서 추출 → 정제 → 적재 작업을 코드로 관리, Dagster로 이관 준비 중.
  • Prometheus에 주요 크롤링 지표(etl_success_total, records_per_run)를 전송 후 Grafana 대시보드로 시각화.
